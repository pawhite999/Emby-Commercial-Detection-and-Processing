CommDetect — Commercial Detection and Processing

Plugin

Full path, e.g. /usr/local/bin/commdetect. The plugin writes ini files to the same directory as this binary.

commdetect.ini — [General]

Detection quality preset. fast skips logo detection; accurate enables all detectors at higher cost.
Scoring window size in seconds.
Frames per second extracted for analysis.
Maximum parallel detector threads. 0 = use all CPU cores.

commdetect.ini — [Emby]

e.g. http://localhost:8096

commdetect.ini — [Watch]

Wait this many seconds after a file stops growing before processing it.
Comma-separated, e.g. .ts,.mpg,.mp4,.mkv
Comma-separated glob patterns to ignore, e.g. *.partial.*,*temp*

commdetect.ini — [Detection]

Combined detector score required to classify a segment as commercial (0–1). Lower = more sensitive.
Ignore this many seconds at the start of the recording (matches DVR pre-padding).
Ignore this many seconds at the end of the recording (matches DVR post-padding).
Extra time added around detected break boundaries.

commdetect.ini — [Psip] & [Xds]

[BlackFrame]

Maximum luma value to consider a pixel "black" (0–255). Default: 12.

[SceneChange]

Scene changes per minute to classify a segment as a commercial. Default: 12.

[Silence]

[Logo]

Seconds used to learn the network logo at the start of the recording.
